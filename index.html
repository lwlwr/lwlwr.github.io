基于 Qwen2.5-3B-Instruct 的单卡强化学习推理能力训练与性能调优

单卡 GRPO / Countdown 可验证任务 / 显存与训练稳定性优化
核心结果：4.3% → 60.4%
格式合法率：87.3% → 99.7% 显存峰值：40GB+ → 24GB 内 RTX 4090 24GB 单卡可复现
  • 独立完成 `Qwen2.5-3B-Instruct` 在单卡 `RTX 4090` 上的 GRPO 训练链路搭建，覆盖数据预处理、奖励函数设计、采样配置、训练监控与评测闭环。
  • 围绕平均奖励、成功率、回复长度、格式奖励等指标重构训练诊断流程，定位输出长度漂移与格式异常问题，并通过分组奖励归一化等方式提升训练可控性。
  • 面向单卡显存约束，系统分析 actor、reference、rollout 等模块的开销，通过优化器参数转移到 CPU、响应长度控制、unfinished episode filtering 等策略将训练压缩进 `24GB`。
  • 项目价值不只是把脚本跑通，而是在受陒资源下建立一套可信的 RL 训练与稳定性优化方法论。

基于 Llama-8B 的 Agentic Research 单卡强化学习训练与优化

Agentic RL / 自举式问答生成 / 检索与答案验证闭环
核心结果：23% → 59%
工具调用格式成功率：+27.7% 100 步训练取得显著收益 8B 模型单卡低成本实验
  • 在 `Llama3.1-8B` 基础上搭建面向领域文档库的 Agentic RL 训练闭环，包含问答生成、语义检索、答案验证与策略优化等关键环节。
  • 将原始“正确性 + 格式奖励”的单一目标扩展为“答案正确性 + 工具调用格式 + 查询改写质量 + 幻觉惩罚”的多目标奖励体系，显著提升回答质量与工具调用稳定性。
  • 针对检索训练与泛化评估一致性问题，升级检索命中记录和数据切分方式，避免训练集与测试集共享相邻文档片段，提升实验结果可信度。
  • 采用 `QLoRA 4-bit`、小批量、短上下文与 Unsloth 优化方案，将 8B 模型 100 步训练时长控制在约 1 小时量级，支撑快速实验迭代。

教育信创平台智能问答模块落地

企业级 RAG / DashScope + LangChain + Milvus/FAISS + FastAPI
核心结果：无依据回答 80% → 5%
试运行问答量：1800+ 清洗语料：3842 条 检索片段：8716 个
  • 参与教育部教育管理信息中心国产化适配需求下的智能问答模块建设，在真实业务场景中完成问答能力接入与验证。
  • 负责知识库构建与检索链路优化，持续迭代分块策略、Top-K、相似度阈值与系统提示词约束，显著降低无依据回答比例。
  • 将 RAG 能力封装为 `/ask`、`/rebuild` 等 REST 服务并接入 Java 主平台，补齐日志监控、异常恢复与向量库迁移预案。
  • 项目让我形成了“算法效果必须通过业务指标证明”的工程意识，而不是只停留在离线实验层面。
